# Cafetera — Система управления столовой

## 📖 Описание проекта

Cafetera — это веб-приложение для управления меню столовой и автоматизации процесса заказа еды. Система позволяет администраторам загружать ежедневное меню, а пользователям — просматривать блюда, фильтровать их по различным параметрам и использовать умный подбор блюд на основе искусственного интеллекта.

---

## 🎯 Функциональные возможности

### Для пользователей

#### 1. Просмотр меню
- **Главная страница** — отображение всех доступных блюд текущего дня
- **Категории блюд** — фильтрация по типам (салаты, супы, горячее, гарниры, напитки, десерты)
- **Временные диапазоны** — выбор периода питания (завтрак, бизнес-ланч, ужин)
- **Детальная информация** — каждое блюдо содержит:
  - Название и описание
  - Состав (ингредиенты)
  - КБЖУ (калории, белки, жиры, углеводы)
  - Цену (если указана)

#### 2. Фильтрация блюд
- **По категориям** — выбор одного или нескольких типов блюд
- **По времени** — завтрак, обед, ужин, бизнес-ланч
- **По КБЖУ** — фильтрация по калорийности и содержанию нутриентов
- **По составу** — исключение нежелательных ингредиентов

#### 3. Умный подбор блюд (Сбор подноса)
- **AI-ассистент** — введите текстовое описание желаемого обеда
- **Контекстный подбор** — система учитывает:
  - Доступные блюда в меню
  - Временной диапазон
  - Правила бизнес-ланча (выгодный подбор)
  - Разнообразие категорий
  - Оптимальное количество блюд (до 3 по умолчанию)
- **Примеры запросов**:
  - "Хочу что-нибудь легкое и полезное"
  - "Нужно сытное мясное блюдо с гарниром"
  - "Предложи обед из трех блюд"
  - "Хочу рыбу с овощами"

#### 4. Корзина
- **Добавление блюд** — выбор понравившихся позиций
- **Просмотр состава** — детальная информация о каждом блюде
- **Подсчет итогов** — автоматический расчет общей стоимости и КБЖУ

### Для администраторов

#### 1. Загрузка меню
- **Формат** — JSON-файл со структурированными данными
- **Namespace** — уникальный идентификатор точки питания (например, "cafeteria-main")
- **Дата** — выбор даты для загружаемого меню
- **Автоподсказки** — при вводе namespace система предлагает существующие значения
- **Проверка** — индикация создания нового namespace

#### 2. Управление точками питания
- **Мульти-namespace** — поддержка нескольких столовых в одной системе
- **История меню** — хранение меню по датам для каждой точки

## 👤 Как добавить администратора

Администраторы хранятся в базе данных PostgreSQL. Для добавления нового администратора используйте скрипт `add_admin.py`:

```bash
# Локально (виртуальное окружение должно быть активировано)
cd backend
source venv/bin/activate
python add_admin.py --login=<username> --password=<password>

# Через Docker
docker-compose exec -T backend python backend/add_admin.py --login=<username> --password=<password>

# Через Makefile
make add-admin LOGIN=<username> PASSWORD=<password>
```

**Пример:**
```bash
make add-admin LOGIN=admin PASSWORD=SecurePass123
```

## 🛠 Техническая реализация

### Стек технологий

#### Backend
- **Язык**: Python 3.12+
- **Фреймворк**: FastAPI (асинхронный веб-фреймворк)
- **ORM**: SQLAlchemy 2.0 (async)
- **База данных**: PostgreSQL 15+
- **Асинхронный драйвер**: asyncpg
- **Валидация**: Pydantic v2
- **HTTP-клиент**: httpx (для вызова LLM API)
- **Сервер**: Uvicorn (ASGI)

#### Frontend
- **Фреймворк**: React 18+
- **Сборщик**: Vite
- **Стилизация**: CSS Modules
- **HTTP-клиент**: Axios
- **Роутинг**: React Router DOM

#### Инфраструктура
- **Контейнеризация**: Docker + Docker Compose
- **Переменные окружения**: python-dotenv
- **AI/LLM**: Внешний API (OpenAI-совместимый)

### Формат JSON для загрузки меню

```json
{
  "namespace": "cafeteria-main",
  "date": "2025-01-15",
  "dishes": [
    {
      "id": 1,
      "name": "Салат Цезарь",
      "category": "salads",
      "composition": "Куриное филе, салат ромэн, пармезан, сухарики, соус цезарь",
      "kkal": 250,
      "proteins": 20,
      "fats": 15,
      "carbs": 10,
      "price": 350,
      "timeRange": {
        "from": "12:00",
        "to": "16:00"
      }
    }
  ],
  "businessLunch": {
    "price": 550,
    "items": [
      {
        "id": 101,
        "name": "Борщ с говядиной",
        "category": "soups",
        "composition": "Говядина, свёкла, капуста, картофель, морковь, лук",
        "kkal": 180,
        "proteins": 12,
        "fats": 8,
        "carbs": 15,
        "timeRange": {
          "from": "12:00",
          "to": "16:00"
        }
      }
    ]
  }
}
```

**Поля блюда:**
- `id` — уникальный идентификатор (число)
- `name` — название блюда (строка)
- `category` — категория (salads, soups, main, sides, drinks, desserts)
- `composition` — состав/ингредиенты (строка)
- `kkal` — калории на 100г (число)
- `proteins` — белки на 100г (число)
- `fats` — жиры на 100г (число)
- `carbs` — углеводы на 100г (число)
- `price` — цена в рублях (число, опционально)
- `timeRange` — временной диапазон доступности (объект с from/to)

### База данных

#### Схема
- **Menu** — меню по датам и namespace
  - `namespace` (VARCHAR, primary key часть)
  - `date` (DATE, primary key часть)
  - `menu_json` (JSONB) — полное меню в JSON-формате

- **Admin** — администраторы
  - `login` (VARCHAR, primary key)
  - `password_hash` (VARCHAR) — хеш пароля (bcrypt)

#### Миграции
На данный момент миграции не используются. Схема создаётся автоматически при первом запуске через `Base.metadata.create_all()`.

Для будущих изменений рекомендуется добавить:
- Alembic для управления миграциями
- Версионирование схемы БД

### LLM Integration

Система использует внешний LLM API для умного подбора блюд.

**Принцип работы:**
1. Фронтенд отправляет пользовательский запрос (например, "хочу лёгкий обед")
2. Бекенд получает актуальное меню из БД
3. Формируется системный промпт со списком доступных блюд (ID, название, КБЖУ, состав)
4. Отправляется запрос к LLM API
5. LLM возвращает массив ID рекомендованных блюд
6. Бекенд возвращает ID фронтенду для отображения

**Правила подбора (в системном промпте):**
- Не более 3 блюд (если пользователь не хочет много)
- Разнообразие категорий (не только супы или только напитки)
- Бизнес-ланч: либо все блюда, либо ни одного (выгодно для пользователя)
- Учёт пожеланий по калорийности и составу

- **Пароли администраторов** — хешируются через bcrypt

- **Валидация входных данных** — Pydantic модели
- **.env файл** — не коммитится в git
